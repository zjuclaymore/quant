# 版本2修改说明（单因子回测）

更新时间：2026-04-11

## 本次目标
- 恢复并默认启用：去极值、标准化、中性化
- 修复分组池与交易池口径不一致问题（方案A）
- 同步备份脚本，防止回滚到旧逻辑
- 更新报告流程文案，避免误导

## 代码修改

### 1) 分组与交易口径统一（方案A）
文件：back_test/sigle_factor_test/src/bt_core.py

- 新增 `_get_strict_liq_symbols(t_buy)`：统一计算严格流动性股票集合
  - `valid_days_20 >= 15`
  - `avg_vol_20 >= 当期截面10%分位`
- `_apply_stock_pool_filter()` 改为调用该统一函数，作为分组前过滤口径
- `_run_trading_loop()` 买入前增加同样过滤，并对 `buy_mask` 二次收敛到同一集合

结果：分组样本与交易买入样本在流动性口径上保持一致，避免“分组可入、交易不可入”的口径漂移。

### 2) 恢复并启用三步因子处理
文件：back_test/sigle_factor_test/src/bt_core.py

- 恢复并启用：
  - 截面 3MAD 去极值 + Z-Score 标准化
  - 行业 + 市值中性化（残差）
- 分组目标列切换为：`{factor_name}_neu`
- 为中性化返回值增加兼容处理（Series/tuple）

结果：分组使用中性化残差，不再直接使用原始因子列。

### 3) 报告流程文案修正
文件：back_test/sigle_factor_test/src/bt_core.py

- `预处理截面` 文案更新为：`3MAD 去极值 + Z-Score 标准化`
- `截面中性化` 文案更新为：`默认开启，行业 + 市值中性化残差`

结果：报告描述与实际执行逻辑一致。

### 4) 备份脚本同步
文件：back_test/sigle_factor_test/src/bt_core.py.bak_before_restore

- 用当前修复后的 `bt_core.py` 全量覆盖备份文件。

结果：后续若从备份恢复，不会回退到“中性化关闭”的旧逻辑。

## 兼容性与风险说明
- 本版本会改变分组输入因子（原始值 -> 中性化残差），历史结果将与版本1不可直接横向比较。
- 由于口径更严格，部分月份可交易样本数可能下降，属于预期变化。

## 建议验证项
- 先跑 1 年窗口（如 2014-01 ~ 2014-12）验证完成链路
- 再跑全量区间
- 对比版本1/版本2：覆盖率、组内样本数、IC/RankIC 与收益曲线差异
